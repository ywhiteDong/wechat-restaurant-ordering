## 🎯 项目概述

### 项目介绍
一个简单实用的微信小程序点餐系统，实现了完整的点餐流程，无需支付接口。

### 核心功能
- ✅ 首页展示（店铺信息、轮播图、热销推荐）
- ✅ 菜单浏览（分类导航、菜品列表、左右联动）
- ✅ 购物车管理（添加/删除商品、数量调整、实时计算总价）
- ✅ 订单确认（用餐方式选择、桌号输入、备注信息）
- ✅ 订单管理（订单列表、订单详情、再来一单）
- ✅ 数据持久化（本地存储）

### 技术栈
- 微信小程序原生框架
- JavaScript
- WXML + WXSS

## 🔧 已解决的问题

### 问题 1：项目导入失败
**错误信息**：
```
Failed to launch simulator
Error: app.json ["pages"][2] could not find the corresponding file
```

**原因**：初次下载时 pages 文件夹内容不完整

**解决方案**：
- 检查并确认所有页面文件完整
- 重新从 GitHub 下载项目

### 问题 2：TabBar 图标缺失
**错误信息**：
```
iconPath 的图片资源 'images/home.png' not found
```

**解决方案**：
修改 `app.json`，移除所有 `iconPath` 和 `selectedIconPath` 配置：

```json
{
  "tabBar": {
    "list": [
      {
        "pagePath": "pages/index/index",
        "text": "首页"
      },
      {
        "pagePath": "pages/menu/menu",
        "text": "点餐"
      },
      {
        "pagePath": "pages/order/order",
        "text": "订单"
      }
    ]
  }
}
```

### 问题 3：pages 配置为空
**错误信息**：
```
pages should not be empty
```

**解决方案**：
确保 `app.json` 中包含完整的 pages 配置：

```json
{
  "pages": [
    "pages/index/index",
    "pages/menu/menu",
    "pages/order/order",
    "pages/orderDetail/orderDetail"
  ]
}
```

### 问题 4：图片加载失败
**错误信息**：
```
Failed to load local image resource /images/banner1.jpg
```

**状态**：警告信息，不影响功能使用

**可选解决方案**：
- 方案 A：暂时忽略（推荐）
- 方案 B：添加对应的图片文件到 images 文件夹

## 🆘 常见问题

### Q1: 重新打开项目后编译失败？

**A**: 尝试以下步骤：
1. 点击顶部"清除缓存"按钮
2. 全选所有选项
3. 点击确定
4. 重新编译

### Q2: 模拟器显示白屏？

**A**: 
1. 检查控制台是否有错误信息
2. 确认 `app.json` 中的 pages 配置正确
3. 尝试重启微信开发者工具

### Q3: 如何在真机上预览？

**A**:
1. 点击顶部"预览"按钮
2. 用微信扫描二维码
3. 在手机上体验

### Q4: 如何发布小程序？

**A**:
1. 确保使用正式 AppID（非测试号）
2. 点击"上传"按钮
3. 填写版本号和描述
4. 登录小程序后台提交审核
5. 审核通过后发布

### Q5: 购物车数据会丢失吗？

**A**: 
- 数据存储在本地，关闭小程序不会丢失
- 但卸载小程序后会丢失
- 如需永久保存，需要连接后端服务器
